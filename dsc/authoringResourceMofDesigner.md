## <a name="creating-resource-properties"></a><span data-ttu-id="00568-112">Создание свойств ресурсов</span><span class="sxs-lookup"><span data-stu-id="00568-112">Creating resource properties</span></span>
<span data-ttu-id="00568-113">В первую очередь необходимо решить, какие свойства будут представлены в ресурсе.</span><span class="sxs-lookup"><span data-stu-id="00568-113">The first thing we need to do is decide on properties that the resource will expose.</span></span> <span data-ttu-id="00568-114">В этом примере мы определим пользователя Active Directory со следующими свойствами.</span><span class="sxs-lookup"><span data-stu-id="00568-114">For this example, we will define an Active Directory user with the following properties.</span></span>

<span data-ttu-id="00568-115">Имя параметра Описание</span><span class="sxs-lookup"><span data-stu-id="00568-115">Parameter name  Description</span></span>
* <span data-ttu-id="00568-116">**UserName**: основное свойство, которое служит уникальным идентификатором пользователя.</span><span class="sxs-lookup"><span data-stu-id="00568-116">**UserName**: Key property that uniquely identifies a user.</span></span>
* <span data-ttu-id="00568-117">**Ensure**: указывает, должна ли учетная запись пользователя присутствовать (Present) или отсутствовать (Absent).</span><span class="sxs-lookup"><span data-stu-id="00568-117">**Ensure**: Specifies whether the user account should be Present or Absent.</span></span> <span data-ttu-id="00568-118">Этот параметр имеет только два возможных значения.</span><span class="sxs-lookup"><span data-stu-id="00568-118">This parameter will have only two possible values.</span></span>
* <span data-ttu-id="00568-119">**DomainCredential**: доменный пароль для пользователя.</span><span class="sxs-lookup"><span data-stu-id="00568-119">**DomainCredential**: The domain password for the user.</span></span>
* <span data-ttu-id="00568-120">**Password**: пароль для пользователя, позволяющий конфигурации при необходимости изменить пароль пользователя.</span><span class="sxs-lookup"><span data-stu-id="00568-120">**Password**: The desired password for the user to allow a configuration to change the user password if necessary.</span></span>

<span data-ttu-id="00568-121">Для создания свойств используется командлет **New-xDscResourceProperty**.</span><span class="sxs-lookup"><span data-stu-id="00568-121">To create the properties, we use the **New-xDscResourceProperty** cmdlet.</span></span> <span data-ttu-id="00568-122">Описанные выше свойства создаются следующими командами PowerShell.</span><span class="sxs-lookup"><span data-stu-id="00568-122">The following PowerShell commands create the properties described above.</span></span>

```powershell
$UserName = New-xDscResourceProperty –Name UserName -Type String -Attribute Key
$Ensure = New-xDscResourceProperty –Name Ensure -Type String -Attribute Write –ValidateSet “Present”, “Absent”
$DomainCredential = New-xDscResourceProperty –Name DomainCredential -Type PSCredential -Attribute Write
$Password = New-xDscResourceProperty –Name Password -Type PSCredential -Attribute Write
```

## <a name="create-the-resource"></a><span data-ttu-id="00568-123">Создание ресурсов</span><span class="sxs-lookup"><span data-stu-id="00568-123">Create the resource</span></span>

<span data-ttu-id="00568-124">Теперь, когда свойства ресурса созданы, можно вызвать командлет **New-xDscResource** для создания ресурса.</span><span class="sxs-lookup"><span data-stu-id="00568-124">Now that the resource properties have been created, we can call the **New-xDscResource** cmdlet to create the resource.</span></span> <span data-ttu-id="00568-125">Командлет **New-xDscResource** выводит список свойств в виде параметров.</span><span class="sxs-lookup"><span data-stu-id="00568-125">The **New-xDscResource** cmdlet takes the list of properties as parameters.</span></span> <span data-ttu-id="00568-126">Кроме того, он принимает путь для создания модуля, имя нового ресурса и имя модуля, в котором он будет храниться.</span><span class="sxs-lookup"><span data-stu-id="00568-126">It also takes the path where the module should be created, the name of the new resource, and the name of the module in which it is contained.</span></span> <span data-ttu-id="00568-127">Ресурс создает следующая команда PowerShell.</span><span class="sxs-lookup"><span data-stu-id="00568-127">The following PowerShell command creates the resource.</span></span>

```powershell

```

<span data-ttu-id="00568-128">Командлет **New-xDscResource** создает MOF-схему, каркас сценария для ресурса, необходимую структуру папок, а также манифест для модуля, предоставляющего новый ресурс.</span><span class="sxs-lookup"><span data-stu-id="00568-128">The **New-xDscResource** cmdlet creates the MOF schema, a skeleton resource script, the required directory structure for your new resource, and a manifest for the module that exposes the new resource.</span></span>

<span data-ttu-id="00568-129">MOF-схема находится в файле **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof** и имеет следующее содержание:</span><span class="sxs-lookup"><span data-stu-id="00568-129">The MOF schema file is at **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.schema.mof**, and its contents are as follows.</span></span>

```
[ClassVersion("1.0.0.0"), FriendlyName("Demo_ADUser")]
class Demo_ADUser : OMI_BaseResource
{
  [Key] string UserName;
  [Write, ValueMap{"Present","Absent"}, Values{"Present","Absent"}] string Ensure;
  [Write, EmbeddedInstance("MSFT_Credential")] String DomainCredential;
  [Write, EmbeddedInstance("MSFT_Credential")] String Password;
};
```

<span data-ttu-id="00568-130">Сценарий ресурса находится в файле **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.psm1**.</span><span class="sxs-lookup"><span data-stu-id="00568-130">The resource script is at **C:\Program Files\WindowsPowerShell\Modules\Demo_DSCModule\DSCResources\Demo_ADUser\Demo_ADUser.psm1**.</span></span> <span data-ttu-id="00568-131">Он не содержит фактической логики реализации ресурса — ее необходимо добавить самостоятельно.</span><span class="sxs-lookup"><span data-stu-id="00568-131">It does not include the actual logic to implement the resource, which you must add yourself.</span></span> <span data-ttu-id="00568-132">Каркас сценария выглядит следующим образом.</span><span class="sxs-lookup"><span data-stu-id="00568-132">The contents of the skeleton script are as follows.</span></span>

## <a name="updating-the-resource"></a><span data-ttu-id="00568-133">Обновление ресурса</span><span class="sxs-lookup"><span data-stu-id="00568-133">Updating the resource</span></span>

<span data-ttu-id="00568-134">Если вам нужно добавить или изменить список параметров для ресурса, используйте командлет **Update-xDscResource**.</span><span class="sxs-lookup"><span data-stu-id="00568-134">If you need to add or modify the parameter list of the resource, you can call the **Update-xDscResource** cmdlet.</span></span> <span data-ttu-id="00568-135">Этот командлет обновляет ресурс с учетом нового списка параметров.</span><span class="sxs-lookup"><span data-stu-id="00568-135">The cmdlet updates the resource with a new parameter list.</span></span> <span data-ttu-id="00568-136">Если логика в сценарий ресурсов уже добавлена, она останется без изменений.</span><span class="sxs-lookup"><span data-stu-id="00568-136">If you have already added logic in your resource script, it is left intact.</span></span>

<span data-ttu-id="00568-137">Предположим, вам нужно включить в ресурс время последнего входа пользователя.</span><span class="sxs-lookup"><span data-stu-id="00568-137">For example, suppose you want to include the last log in time for the user in our resource.</span></span> <span data-ttu-id="00568-138">Вместо того чтобы писать ресурс заново, можно выполнить командлет **New-xDscResourceProperty**, чтобы создать еще одно свойство, а затем командлет **Update-xDscResource**, чтобы добавить его в список свойств.</span><span class="sxs-lookup"><span data-stu-id="00568-138">Rather than writing the resource again completely, you can call the **New-xDscResourceProperty** to create the new property, and then call **Update-xDscResource** and add your new property to the properties list.</span></span>

```powershell
$lastLogon = New-xDscResourceProperty –Name LastLogon –Type Hashtable –Attribute Write –Description “For mapping users to their last log on time”
Update-xDscResource –Name ‘Demo_ADUser’ –Property $UserName, $Ensure, $DomainCredential, $Password, $lastLogon -Force
```
