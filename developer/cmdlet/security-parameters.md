# <a name="security-parameters"></a>Параметры безопасности

Ниже перечислены рекомендуемые имена и функциональные возможности для параметров, используемых для предоставления сведений о безопасности для операции, такие как параметры, указывающие сведения о сертификате ключа и привилегий.

|Параметр|Функции|
|---|---|
|**СПИСОК УПРАВЛЕНИЯ ДОСТУПОМ**<br>Тип данных: Строка|Реализуйте этот параметр для указания уровня управления доступом защиты для каталога или для универсального кода ресурса (URI).|
|**CertFile**<br>Тип данных: Строка|Реализуйте этот параметр, таким образом, чтобы пользователь может указать имя файла, содержащего одно из следующих:<br>-Сертификат x.509 в кодировке Base64 или различающееся правил кодирования (DER)<br>-Public Key Cryptography стандартов (PKCS) #12 файл, содержащий по крайней мере один сертификат и ключ|
|**CertIssuerName**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать имя издателя сертификата или таким образом, пользователь может указать подстроки.|
|**CertRequestFile**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы указать имя файла, который содержит запрос на сертификат Base64 или DER-кодировке PKCS #10.|
|**CertSerialNumber**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы указать серийный номер, который был выдан центром сертификации.|
|**CertStoreLocation**<br>Тип данных: Строка|Реализуйте этот параметр, благодаря которому пользователь может указать расположение хранилища сертификатов. Расположение — обычно это путь к файлу.|
|**CertSubjectName**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать издателя сертификата или таким образом, пользователь может указать подстроки.|
|**CertUsage**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы указать использование ключа или расширенное использование ключа. Ключ может быть представлен как немного маски, немного, идентификатор объекта (OID), или строку.|
|**Учетные данные**<br>Тип данных: [System.Management.Automation.PSCredential](/dotnet/api/System.Management.Automation.PSCredential)|Реализуйте этот параметр, чтобы командлет автоматически пользователю будет предложено ввести имя пользователя или пароль. Текст запроса и отображается в том случае, если полный учетных данных не предоставляется напрямую.|
|**CSPName**<br>Тип данных: Строка|Реализуйте этот параметр, благодаря которому пользователь может указать имя сертификата поставщика услуг (CSP).|
|**CSPType**<br>Тип данных: Целое число|Реализуйте этот параметр, благодаря которому пользователь может указать тип CSP.|
|**Группа**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать коллекцию участников, для доступа. Дополнительные сведения см. в описании **участника** параметра.|
|**KeyAlgorithm**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать алгоритм создания ключа для защиты.|
|**КонтейнерКлючей**<br>Тип данных: Строка|Реализуйте этот параметр, благодаря которому пользователь может указать имя контейнера ключа.|
|**Значение KeyLength**<br>Тип данных: Целое число|Реализуйте этот параметр, чтобы пользователь может указать длину ключа в битах.|
|**Операция**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать действие, которое может быть выполнена на защищенных объектов.|
|**Основной**<br>Тип данных: Строка|Реализуйте этот параметр, таким образом, пользователь может указать уникальную идентифицировать сущность доступа.|
|**привилегий**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать справа, командлет необходимо выполнять операции для конкретной сущности.|
|**Привилегии**<br>Тип данных: Массив привилегий|Реализуйте этот параметр, благодаря которому пользователь может указать права, необходимые для выполнения своих операций для определенной записи для командлета.|
|**Роли**<br>Тип данных: Строка|Реализуйте этот параметр, таким образом, пользователь может указать набор операций, которые могут быть выполнены в сущности.|
|**SaveCred**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы учетные данные, которые ранее были сохранены с пользователя будет использоваться при указании параметра.|
|**Область**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать группы защищаемых объектов для командлета.|
|**ИД БЕЗОПАСНОСТИ**<br>Тип данных: Строка|Реализуйте этот параметр, благодаря которому пользователь может указать уникальный идентификатор, который представляет участника.|
|**доверенные**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы уровни доверия поддерживаются в том случае, если этот параметр указан.|
|**trustLevel**<br>Тип данных: Ключевое слово|Реализуйте этот параметр, чтобы пользователь может указать уровень доверия, который поддерживается. Например возможные значения включают в Интернете, интрасети и полное доверие.|
