# <a name="activity-parameters"></a>Параметры действий

Ниже перечислены рекомендуемые имена и функции для параметров действия.

|Параметр|Функции|
|---|---|
|**добавить**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы пользователь сможет добавлять содержимое в конец ресурс, если указан параметр.|
|**CaseSensitive**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы пользователь может потребовать чувствительность к регистру, при указании параметра.|
|**Команда**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать командную строку для запуска.|
|**CompatibleVersion**<br>Тип данных: Объект System.Version|Реализуйте этот параметр, чтобы пользователь может указать семантику, что командлет должен быть совместим с, для совместимости с предыдущими версиями.|
|**Сжатие**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, таким образом сжатие используется в том случае, если указан параметр.|
|**Сжатие**<br>Тип данных: Ключевое слово|Реализуйте этот параметр, благодаря которому пользователь может указать алгоритм, используемый для сжатия данных.|
|**Непрерывная**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы данные обрабатываются, пока пользователь не закроет командлет при указании параметра. Если этот параметр не указан, командлет обрабатывает предопределенные объем данных и затем завершает операцию.|
|**Создание**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы указать, что ресурс создается, если он уже существует при указании параметра.|
|**Удаление**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы ресурсы будут удалены при командлет завершил свою работу, если указан параметр.|
|**Очистка**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы указать, что невыполненные операции обрабатываются перед командлет обрабатывает новые данные, если указан параметр. Если этот параметр не указан, рабочие элементы обрабатываются немедленно.|
|**Стирание**<br>Тип данных: Int32|Реализуйте этот параметр, таким образом, пользователь может указать количество раз, когда ресурс удаляется перед его удалением.|
|**ErrorLevel**<br>Тип данных: Int32|Реализуйте этот параметр, таким образом, пользователь может указать уровень ошибок в отчет.|
|**исключить**<br>Тип данных: String[]|Реализуйте этот параметр, таким образом, пользователь может исключить что-то из действия. Дополнительные сведения об использовании фильтров входа см. в разделе [входные параметры фильтра](input-filter-parameters.md).|
|**Фильтр**<br>Тип данных: Ключевое слово|Реализуйте этот параметр, чтобы пользователь может указать фильтр, который выбирает ресурсы, на которой требуется выполнить действие командлета. Дополнительные сведения об использовании фильтров входа см. в разделе [входные параметры фильтра](./input-filter-parameters.md).|
|**Выполните**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, таким образом, чтобы ход выполнения отслеживается в том случае, если указан параметр.|
|**Force**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы указать, что пользователь может выполнить действие, даже если ограничения встречаются в том случае, если указан параметр. Параметр не поддерживает безопасность скомпрометировано. Например этот параметр позволяет пользователю перезаписать файл, доступный только для чтения.|
|**включить**<br>Тип данных: String[]|Реализуйте этот параметр, таким образом, пользователь может включать что-нибудь в действии. Дополнительные сведения об использовании фильтров входа см. в разделе [входные параметры фильтра](input-filter-parameters.md).|
|**Добавочное**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы указать, что обработка выполняется постепенно при указании параметра. Например этот параметр позволяет пользователю выполнять добавочное резервное копирование, резервную копию файлов только с момента последнего резервного копирования.|
|**InputObject**<br>Тип данных: Объект|Этот параметр следует использовать, когда командлет получает входные данные из других командлетов. При определении **InputObject** параметра, всегда указывайте **ValueFromPipeline** ключевое слово при объявлении **параметр** атрибута. Дополнительные сведения об использовании фильтров входа см. в разделе [входные параметры фильтра](./input-filter-parameters.md).|
|**Вставить**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет вставляет элемент, если указан параметр.|
|**Интерактивный**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет работает интерактивно с пользователем, если указан параметр.|
|**интервал**<br>Тип данных: Хэш-таблицы|Реализуйте этот параметр, чтобы пользователь может указать ключевые слова хэш-таблицу, содержащую значения. В следующем примере показано образцы значений для **интервал** параметр: `-interval @{ResumeScan=15; Retry=3}`.|
|**Log**<br>Тип данных: SwitchParameter|Реализуйте этот параметр аудита действия командлета, при указании параметра.|
|**Параметр NoClobber**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы ресурс не перезаписывается, когда этот параметр указан. Этот параметр обычно относится к командлетов, которые создают новые объекты, чтобы они могут быть защищены от перезаписи существующих объектов с одинаковым именем.|
|**Notify**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы пользователь уведомляется о завершении действия, если указан параметр.|
|**NotifyAddress**<br>Тип данных: Адрес электронной почты|Реализуйте этот параметр, чтобы пользователь может указать адрес электронной почты, используемый для отправки уведомлений при **Notify** указан параметр.|
|**Перезаписать**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, таким образом, чтобы командлет перезаписывает все существующие данные при указании параметра.|
|**Запрос**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать запрос для командлета.|
|**Скрытый**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет отключает отзывы пользователей во время его действия, если указан параметр.|
|**Recurse**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы рекурсивно командлет выполняет свои действия на ресурсы, когда этот параметр указан.|
|**Восстановление**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет попытки исправить чего-либо из поврежденном состоянии, если указан параметр.|
|**RepairString**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать строку, используемую при **восстановления** указан параметр.|
|**Повторите попытку**<br>Тип данных: Int32|Реализуйте этот параметр, чтобы пользователь может указать, сколько раз, командлет попытается действие.|
|**Выберите**<br>Тип данных: Ключевое слово массива|Реализуйте этот параметр, благодаря которому пользователь может указать массив типов элементов.|
|**поток**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы пользователю осуществлять потоковую передачу нескольких выходных объектов по конвейеру, при указании параметра.|
|**Strict**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы все ошибки будут обрабатываться как прерывающие ошибки, если указан параметр.|
|**TempLocation**<br>Тип данных: Строка|Реализуйте этот параметр, чтобы пользователь может указать расположение временных данных, который используется во время операции командлета.|
|**время ожидания**<br>Тип данных: Int32|Реализуйте этот параметр, благодаря которому пользователь может указать интервал времени ожидания (в миллисекундах).|
|**усечение**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет будет выполнять усечение его действия, если указан параметр. Если этот параметр не указан, командлет выполняет другое действие.|
|**Проверка**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет будет проверить, чтобы определить, произошло ли действие, если указан параметр.|
|**Подождите**<br>Тип данных: SwitchParameter|Реализуйте этот параметр, чтобы командлет будет ожидать ввода пользователя, прежде чем продолжить, если указан параметр.
|**Время ожидания**<br>Тип данных: Int32|Реализовать этот параметр, благодаря которому пользователь может указать продолжительность (в секундах), командлет будет ожидать пользователя входных данных при **ожидания** указан параметр.|
